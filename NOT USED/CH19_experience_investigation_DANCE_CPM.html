Memory trick: Detectives solve a case — you solve the experience. The first five letters DANCE describe the investigation itself; CPM is what you do with the answer afterwards. Monitoring is the epilogue that always gets marks.

Three natural phases — easier to recall under exam pressure than 8 scattered bullets.

Phase 1 · Run the investigation (DANCE)
D Data collection & cleaningGather policy records, claims, deaths. Check for duplicates, missing data, IBNR/IBNER. Choose investigation period — recent enough to be relevant, long enough to be credible.
A Analyse into homogeneous cellsSubdivide by age, gender, policy duration, smoker status, care level (L1/L2), ADL score, territory, sales channel. Each cell must be genuinely homogeneous.
N Normalise — calculate exposed-to-riskCentral or initial exposed-to-risk. Account for partial years, withdrawals, new entrants. For INLTC: only count time in correct benefit level for each cell.
C Compare actual deaths vs expectedChoose a standard table (e.g. CMI INLTC IL series) or the original pricing basis. Compute A/E ratios per cell. Identify cells with materially different experience.
E Extrapolate trendsLook at own experience over a longer window. Cross-reference industry and population data (CMI, ONS). Identify persistent trends — improvement, deterioration, care pathway changes.
Phase 2 · Set the assumption (CPM)
C Calibrate the basisDerive best-estimate rates from A/E ratios. Smooth or graduate sparse cells. For new products (e.g. enhanced L1) use analogous data, duration proxies, or industry tables with adjustment.
P Project future ratesApply mortality improvement factors. Allow for future changes — underwriting, care pathways, medical advances, population ageing. Extrapolate recent experience trends.
M Margin for uncertaintyAdd prudence (lighter mortality for annuities = lower expected deaths = longer payment). Larger margin where data is thin or the product is novel. State the margin explicitly.
🔍 INLTC-specific complications to add
⏱ Duration effect (select period)INLTC lives are sickest at inception. Mortality falls then rises — subdivide by policy year. Treating all durations alike masks the select effect.
🔀 Benefit level leakage (L1 → L2)L1 exposed-to-risk is contaminated as lives deteriorate to L2 over time. Fix: track address changes, use early durations only, or apply a duration-based proxy.
🔄 Monitor & update — alwaysEspecially for the new variant. Close monitoring is imperative; update pricing basis as experience emerges. This always earns marks.
